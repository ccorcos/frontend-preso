Function Currying

What: functions that take arguments one at a time.

                      
  const add = x => y => x + y

  const add1 = add(1)
  const add2 = add(2)

  add1(10) // 11
  add2(10) // 12

Function Currying

Why: function composition

                      
  const pipe = fns => value =>
    fns.reduce((f,v) => f(v), value)

  pipe([
    add(1),
    times(2),
    minus(3),
  ])(4) // 7

Function Currying

Why: point-free programming

                      
  const pipe = fns => value =>
    fns.reduce((f,v) => f(v), value)

  const computation = pipe([
    add(1),
    times(2),
    minus(3),
  ])

  // computation = x => {
  //   const a = add(1)(x)
  //   const b = times(2)(a)
  //   const c = minus(3)(b)
  //   return c
  // }

  computation(4) // 7
  computation(5) // 9

Purity and Immutablility

What:

  • a pure function returns the same output for the same input
  • data should never be mutated "in-place" (with the same reference pointer)

Purity and Immutablility

Why:

  • easier to statically analyze
  • easier to understand

Purity and Immutablility

Example:

                      
  var xs = [1, 2, 3, 4, 5]

  // pure
  xs.slice(0, 3)
  //=> [1, 2, 3]

  xs.slice(0, 3)
  //=> [1, 2, 3]

  xs.slice(0, 3)
  //=> [1, 2, 3]

Purity and Immutablility

Counter-example:

                      
  var xs = [1, 2, 3, 4, 5]

  // impure
  xs.splice(0, 3)
  //=> [1, 2, 3]

  xs.splice(0, 3)
  //=> [4, 5]

  xs.splice(0, 3)
  //=> []

Persistent Data Structures

What: performant immutable updates

Persistent Data Structures

Why:

  • cloning objects is inefficient
  • garbage thrashing

Persistent Data Structures

Example: a linked-list!

                      
  const append = value => list => ({
    value,
    next: list
  })

  const list1 = {value: 1, next: undefined}
  const list2 = append(2)(list1)

  // "persisted" the previous data structure
  list2.next === list1 // true

What:

Persistent Data Structures

Example: an ImmutableJS Map

                      
  import { Map } from 'immutable'

  const person = new Map({
    name: 'chet',
    age: 25,
    profile: new Map({
      homepage: 'https://chetcorcos.com'
    })
  })

  const next = person.update('age', x => x + 1)

  person.get('age') // 25
  next.get('age') // 26
  person.get('profile') === next.get('profile') // true

Redux

"A predictable state container for JavaScript apps"

  • views and logic are all pure functions
  • global event bus
  • state machine using a reducing function

Redux

Concept:

                      
  import { createStore } from 'redux'

  const initialState = { count: 0 }
  const reducer = (state=initialState, action) => {
    switch(action.type) {
      case 'increment':
        return {
          ...state,
          count: state.count + 1,
        }
    }
  }

  const store = createStore(reducer)

  store.getState() // { count: 0 }
  store.dispatch({type: 'increment'})
  store.getState() // { count: 1 }
